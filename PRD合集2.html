AI 健康管理助手 --- 产品需求文档（PRD）

版本：v5.02026-07-12状态：已审核通过产品形态：iOS / Android App（可选配智能耳夹硬件）

1. 执行摘要

AI 健康管理助手是一款面向全年龄段用户的健康管理 App，同时为男性和女性提供深度健康管理。核心差异化在于两层设计：

第一，Agent 主动推理层。通过基本档案到每日建议四步闭环，结合可穿戴设备数据、天气 API、体检报告解析，提供个性化健康管理。

第二，Agent 实体化为橡皮糖形态的电子宠物，可悬浮附着在首页任意组件上，根据上下文自动移动到需关注的模块上方，支持点击对话和长按语音。

产品内置运动记录功能，涵盖 50+ 运动类型、GPS 轨迹追踪、实时心率区间、运动后恢复分析。同时支持接入 Apple Health / Garmin 等平台补充数据。

首期覆盖六个通用模块（睡眠管理、心率管理、压力管理、体温管理、运动管理、补剂管理）以及两套深度管理模块（女性健康、男性健康）。所有指标按性别和年龄自动配置。

2. 产品定位

2.1 一句话定位

一个会主动管理你健康的 AI 电子宠物。不是让你看数据的仪表板，而是替你记住该做什么的伙伴。

2.2 不是什么

不是说明
不是 AI 问答机器人不追求你问我答，而是主动推算、主动提醒、主动管理
不是纯运动追踪 App运动记录是基础能力，不堆砌运动数据功能
不是女性专用 App女性和男性的深度健康管理同等投入
不是医疗诊断工具所有建议仅供参考，不做疾病诊断

2.3 为什么是我们

市场机会产品方案
用户每天收到大量健康数据但不知道该怎么办Agent 主动给出每日 1-3 条关键建议
女性健康需求（周期/备孕/更年期）被穿戴设备忽视女性专属 6 项深度指标，按年龄自动配置
男性健康需求（呼吸暂停/前列腺/HRV 衰退）无人覆盖男性专属 5 项深度指标，40 岁+自动开启
健康 App 交互冰冷，难以坚持橡皮糖电子宠物，会移动、有情绪、主动找你

3. 需求分析

3.1 问题背景

可穿戴健康管理市场存在四个结构性缺口：

缺口一：数据有了，但洞察没有。用户每天收到几十个数字，但绝大多数 App 无法回答这些数字意味着什么。少数竞品（Whoop Coach、Oura Advisor）开始提供 AI 数据问答，但均为被动模式（用户提问 -> AI 回复），而非主动推送和管理。

缺口二：设备默认用户是男性。女性的周期波动、围绝经期变化在算法层面被忽略。同时男性的睡眠呼吸暂停、前列腺早期信号等也未得到深度覆盖。

缺口三：健康管理需要主动行为。用户需要的是"你记得提醒我"，而不是"等你打开才看到"。

缺口四：健康 App 缺乏情感温度。一个会主动来找你的电子宠物，比冰冷的数据面板更能帮助用户坚持。

3.2 目标用户

用户群体核心痛点产品侧重点
25-40 岁职场女性周期与压力叠加，找不到关联规律周期追踪 + PMS 预警 + 压力管理
40-55 岁围绝经期女性潮热、睡眠紊乱，缺乏量化工具潮热检测 + 更年期综合管理

3.4 用户场景

场景一：晨间简报（v1.2+） 早上 7:30。橡皮糖蹲在简报卡片上："昨晚睡了 7.2h，恢复评分 82。今天卵泡期后期，适合力量训练。室外 26C，补水 1.8L。铁剂该吃了。"

场景二：补剂催促（v1.3+） 下午 3 点补剂未确认。用户打开 App，橡皮糖已跳到补剂卡片上快速弹跳。确认服用后橡皮糖跳回默认位变开心表情。

场景三：HRV 下降预警（v1.2+） 橡皮糖蹲在心率管理卡片上，颜色暖橙。点开看到："HRV 连续 3 天下降 22%，建议完全休息。运动强度已自动下调。"

场景四：运动记录（v1.4+） 选择户外跑步。GPS 追踪轨迹，心率实时显示。结束后 Agent 推送：恢复评分 78，建议补蛋白。

场景五：语音对话（v2.0+） 长按橡皮糖问"今天状态怎么样"，Agent 语音回复。此功能在 Phase 2 上线。

4.4 策略选择

1. 不做问答式 AI。方向是主动管理 + 实体化交互。 2. 不做纯运动 App。运动记录是基础能力。 3. 女性和男性健康同等投入。 4. 补剂管理 + 体检报告解析是本产品独有的功能组合。 5. 不是技术领先，而是交互范式领先——规则引擎做确定性推理，橡皮糖做情感连接。

5.3 橡皮糖电子宠物（核心交互概念）

Agent 不是固定图标，而是可自由活动的电子宠物。它根据当前状态选择合适组件"蹲"在上面，用户打开 App 一眼就能看到哪里需要关注。

传统 App本产品
小红点/角标橡皮糖直接跳到模块上蹲着
先看通知再找模块打开 App 一眼看到哪里有异常
Agent 固定位置自由附着，与界面融合
交互是事务性的有温度的交互：它来找你，你点它

交互方式：点击进入文字对话界面，长按触发语音对话（v2.0）。橡皮糖有 9 种状态（待机/有话说/担心/开心等），不同状态对应不同颜色和动画表现。具体设计细节参考 Demo。

5.5 用户故事

US-01 基本档案注册 As a 新用户，I want to 输入性别/年龄/基础健康信息，So that Agent 能设置正确的监测指标和基线。 验收条件：必填性别/年龄/身高/体重；可选 5 题健康问卷；可选上传体检报告（AI 即时解析）；完成后 Agent 开始个性化配置。

US-02 每日简报推送 As a 活跃用户，I want to 每天早上收到 Agent 生成的简短健康简报，So that 我不自己分析数据就知道今天注意什么。 验收条件：清晨推送（6:00-8:00）；融合睡眠+天气+周期+恢复评分；显示 1-3 条建议；打开率 > 65%。

US-03 补剂提醒与确认 As a 有补剂需求的用户，I want to 设置补剂时间和剂量到点收到提醒，So that 我不会忘记吃。 验收条件：可添加补剂（名称/剂量/时间/服用方式）；到点推送；点击即确认（无需打开 App）；连续 3 次确认后减频率；连续 3 次忽略后主动询问调时间；日历视图记录。

US-04 运动记录 As a 运动用户，I want to 通过 App 记录运动，So that 能追踪运动数据和恢复状态。 验收条件：支持 5+ 种运动类型（v1.4）/ 50+ 种（v3.2）；GPS 记录轨迹；实时心率区间；运动后报告；Agent 恢复建议。

US-05 女性周期追踪 As a 女性用户，I want to 查看生理周期阶段和基础体温，So that 了解周期规律和排卵窗口。 验收条件：自动识别周期阶段；显示 BBT 曲线并标注阶段；备孕模式显示排卵预测；PMS 预警。

US-06 男性呼吸质量筛查 As a 35 岁+男性用户，I want to 查看夜间呼吸质量评估，So that 了解睡眠呼吸暂停风险。 验收条件：AHI 估算值；分级标签（正常/轻度/中度/重度）；打鼾时长强度；40 岁+额外显示夜起趋势。

US-07 Agent 异常预警 As a 用户，I want to 指标偏离基线时收到 Agent 主动预警，So that 及时关注身体变化。 验收条件：HRV 连续 3 天下降 > 15% 触发；静息心率连续 3 天上升 > 5bpm 触发；橡皮糖自动移动到对应模块转为担心状态。

US-08 运动疲劳预警 As a 运动用户，I want to Agent 根据恢复状态建议训练强度，So that 不会过度训练。 验收条件：综合 HRV/静息心率/深睡判断；恢复不足时显示预警；含具体数据和建议；女性自动扣除周期影响。

US-09 语音交互（v2.0） As a 忙碌的用户，I want to 长按橡皮糖用语音提问，So that 不需要打字。 验收条件：长按 350ms 收音；松手发送；TTS 回复；23:00-7:00 切换文字模式。

5.6 首次启动流程

三步设置：基本档案（性别/年龄/身高/体重/问卷 + 可选上传体检报告）-> 目标选择（8 个方向选 1-3 个）-> 数据接入（Apple Health / Google Fit 或跳过）。次日早上 Agent 送出第一条简报。

1. 为什么选择耳夹形态

1.1 耳部的生理信号优势

智能穿戴设备的传感器位置直接影响信号质量。耳部在多个关键生命体征的采集上具有不可替代的优势：

PPG 信号质量优于手腕

光电容积描记法（PPG）是心率、HRV、血氧的核心传感技术。耳垂的组织厚度较薄（3-5mm），毛细血管密度高，血流灌注稳定，且耳部相对心脏的位置固定。相比之下，手腕会随手臂摆动高于或低于心脏水平，导致静脉回流变化和运动伪影，显著降低 PPG 信号的信噪比。

已有研究表明，耳部 PPG 在运动状态下的信号可用率比手腕 PPG 高 30-50%（参考：Tamura et al., Progress in Biomedical Engineering, 2019）。

体温接近核心温度

鼓膜温度是临床公认的核心体温替代测量点。耳夹虽然在耳垂而非耳道内，但其位置仍比手腕远端更接近核心体温源。腕部皮肤温度受外周血管收缩和环境温度的影响极大——低温环境下手腕温度可偏离核心体温 3-5C，而耳部偏离通常小于 1C。

声学通道优势

耳部是唯一可以自然拾取体内声学信号的位置。心音（S1/S2）、呼吸音、鼾声等声学生理信号在耳道范围内清晰可采集，而这些信号手腕设备完全无法获取。

头部运动的独特性

睡眠阶段识别依赖加速度计数据。手臂在睡眠中会有大量非移动性抖动（翻身时的惯性摆动），而头部运动更直接地反映真实的身体姿势变化和起床动作，使夜起检测和睡眠分期的准确率更高。

维度本产品相比竞品的优势本产品的劣势

2.3 竞品启示

1. 不做手表替代品。 耳夹不试图提供 GPS、ECG、蜂窝网络等手表核心功能，而是聚焦健康传感精度和佩戴舒适度。 2. 声学传感是护城河。 鼾声检测、心音分析、呼吸音监测是腕戴和指戴产品无法复制的传感维度，应作为核心卖点。 3. 舒适度决定佩戴率。 重量越轻、佩戴越无感的设备，用户佩戴率越高，数据连续性越好。耳夹的 < 3g 是相比手表手环的核心优势。
